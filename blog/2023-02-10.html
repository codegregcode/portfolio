When I last posted I was just starting the Backend module with Command Shift, and at that point was part way through an exercise using SQL and pgAdmin.

Learning SQL meant a lot of retrieving data, updating and adding data to databases, joining databases, and to be brutally honest I found it a bit boring (even though I know it'll have it's uses). So, I was really pleased when we started learning how to create our own APIs using express JS!

So most of my time this has been spent creating an API that works with the code I made way back at the start of the course when first learning JavaScript. There's a strange sense of satisfaction when I send a get and post request to my server and then receiving the (correct) response that my code has generated! Much more exciting than SQL!

You can see an example of the API working above. To explain in a bit more detail, what I'm doing is sending a 'get' request with the parameters (numbers to add or subtract) in the url. Also, please feel free to look over the code and even fork/clone over on the GitHub page

The first image shows the functions being imported from /lib. In the second image you'll see how the express app responds to the request. On lines 57 and 58 the variables int1 and int2 take the parameters :num1 and :num2 (which are currently strings) and turn them into integers. After from lines 60 to 63 we check whether int1 or int2 is not a number. isNaN returns true if not a number and using a ternary ? expression we say "well if either return true (meaning one or both is not a number), then we're going to set teh status to error code 400, and send JSON displaying the error message...however...if false...and they're both numbers, then we'll set status to 200 (ok) and send JSON with the function needed passing in int1 and int2 as the arguments required. "

You can see how in the post request we send the arguments in the body as JSON.
